`policy.yaml` holds named configuration values for a flow. They are referenced in CEL as
`policy.*` and in prompts as `{{variable}}`.

```yaml policy.yaml
sla_hours: 24
max_escalations: 3
```

## Inheritance

Policy is hierarchical. A child flow inherits root and parent policy and overrides specific
keys. Guards and prompt templates resolve a key by walking child to parent to root.

```yaml
# parent policy.yaml
timeout_hours: 24
max_retries: 3

# child policy.yaml
timeout_hours: 48    # overrides the parent; max_retries stays 3
```

## Budget thresholds

Spend monitoring is configured here as percentages. Crossing one emits
`platform.budget_threshold_crossed` with a matching `level`. If these keys are absent, budget
monitoring is disabled.

```yaml
budget_warning_percent: 60
budget_throttle_percent: 80
budget_emergency_percent: 95
```

See [Budget and cost](/operate/budget-and-cost).

## MCP servers

```yaml
mcp_servers:
  postgres:
    transport: http        # http or stdio
    url: "https://mcp.internal/pg"
    prefix: pg             # required, unique across servers
    credentials_key: pg_token
```

See the [MCP gateway reference](/reference/mcp-gateway).
