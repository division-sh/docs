`schema.yaml` declares a flow's public surface: its lifecycle, the pins other flows wire to,
and the agent roles it requires.

## State machine

<ParamField path="initial_state" type="string">
  The state a new entity starts in. Set to `null` for a stateless flow.
</ParamField>
<ParamField path="states" type="list">
  All states in the flow. Empty for a stateless flow.
</ParamField>
<ParamField path="terminal_states" type="list">
  Absorbing states. Once an entity reaches one, new events for it are rejected and its timers
  are cancelled.
</ParamField>

## Pins

Pins are the flow's public interface. Event pins declare what it accepts and emits; data pins
declare which entity fields it reads and writes.

```yaml
pins:
  inputs:
    events: [order.created]
    reads: [priority]
  outputs:
    events: [order.fulfilled, order.cancelled]
    writes: [status, total]
```

No two flows may write the same data pin (a boot error). An emit of a pin-declared output
event must resolve a target or set `broadcast: true`; see
[Events and routing](/concepts/events-and-routing).

## Required agents

<ParamField path="required_agents" type="list">
  The roles the flow needs. Each entry has `role`, `subscribes_to`, `emits`, and an optional
  `description`.
</ParamField>

Fulfillment is checked at boot on two conditions: an agent in `agents.yaml` whose map key
matches the `role`, and whose subscriptions and `emit_events` cover the declared
`subscribes_to` and `emits`.

## Other fields

<ParamField path="instance_variables" type="object">
  Variables populated at flow-instance creation via `config_from`, available to prompts as
  `{{variable}}`.
</ParamField>
<ParamField path="auto_emit_on_create" type="object">
  For template flows: an event the platform emits once a new instance is fully started.
</ParamField>

## Example

```yaml schema.yaml
initial_state: new
terminal_states: [fulfilled, cancelled]
states: [new, validated, fulfilling, fulfilled, cancelled]
pins:
  inputs:
    events: [order.created]
  outputs:
    events: [order.fulfilled, order.cancelled]
required_agents:
  - role: validator-agent
    subscribes_to: [order.created]
    emits: [order.validated]
```
