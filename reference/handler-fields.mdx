A handler's fields execute in a fixed dependency graph, not source order, and the whole
handler commits in one transaction. This page catalogs every field. For the conceptual model
see [System nodes and handlers](/concepts/system-nodes-and-handlers).

## Execution order

```text
entity acquisition (create_entity | select_entity | select_or_create_entity)
  -> query -> clear_gates -> guard -> accumulate -> filter -> reduce -> count -> compute
    -> branch selection (on_complete / rules)
      -> { advances_to, sets_gate, data_accumulation }   (independent of each other)
        -> projection -> emit.fields -> emit.event -> action -> clear
```

Two short-circuits stop a handler early: a failed `guard` runs its `on_fail` action, and an
incomplete `accumulate` records the arrival and waits (outcome `waiting`). For the precedence
rules (what overrides what, when projection runs, chain-depth and atomicity), see the
[handler execution model](/reference/execution-model).

## Core vs extension fields

**Core** (most handlers): `guard`, `advances_to`, `sets_gate`, `data_accumulation`, `emit`,
`rules`, `on_complete`.

**Extension** (specialized): `accumulate`, `compute`, `fan_out`, `filter`, `reduce`, `count`,
`query`, `clear`, `clear_gates`, `action`.

## Entity acquisition

A stateful input-pin handler declares exactly one acquisition mode (more than one is a boot
error):

<ParamField path="create_entity" type="boolean">
  Mint a fresh entity at `initial_state` before any other step. The new entity gets an
  auto-generated id and the inbound event's `entity_id` is **ignored** for state operations;
  every declared field is materialized (its `initial:` value, else the type default) so guards
  and conditions can read it immediately. Prohibited with `accumulate` (boot error). Allowed
  with `fan_out`: one entity is created and every fanned-out event shares it.
</ParamField>
<ParamField path="select_entity" type="object">
  Resolve exactly one existing flow-owned entity by a declared business key (`by` maps entity
  fields to `payload.*` references). Zero or multiple matches fail closed.
</ParamField>
<ParamField path="select_or_create_entity" type="object">
  Resolve one active match, or deterministically mint one from the declared key.
</ParamField>

## Guard

<ParamField path="guard" type="object">
  A check (or list of checks) evaluated before the handler proceeds. Fields: `id`, `check`
  (a CEL expression), and `on_fail`. `on_fail` is one of `reject` (default), `kill`,
  `discard`, or `escalate:{event}`.
</ParamField>

## Branching

<ParamField path="on_complete" type="list">
  An ordered list of `{condition, advances_to, emit}`. The first matching condition wins.
  Used after accumulation or computation.
</ParamField>
<ParamField path="rules" type="list">
  A list of named branches matched against the payload. Each has `id`, `condition`, and
  optional `emit`, `advances_to`, and `data_accumulation`. The matched rule owns the emit.
</ParamField>

`on_complete` and `rules` are mutually exclusive.

## State and data

<ParamField path="advances_to" type="string">
  Set `entity.current_state` to a target state.
</ParamField>
<ParamField path="sets_gate" type="string">
  Set `entity.gates.{name}` to true.
</ParamField>
<ParamField path="clear_gates" type="boolean">
  Reset gates to false (runs before guard). All-or-nothing, and **entity-wide**: it clears every
  gate on the entity, not just the gates this node declares. Selective clearing is not current.
</ParamField>
<ParamField path="data_accumulation" type="object">
  Write fields to entity state. `writes` is a list; `source_event` defaults to the trigger
  event. Each write item is one of four forms (see below).
</ParamField>

### data_accumulation write forms

```yaml
writes:
  - category                          # direct:   payload.category -> entity.category
  - source_field: pri                 # mapped:   payload.pri -> entity.priority
    target_field: priority
  - target_field: source              # literal:  a constant
    value: "support"
  - target_field: attempts            # computed: a CEL expression
    expression: "entity.attempts + 1"
```

`value` and `expression` are mutually exclusive on one item. The constant key is `value`;
`{target_field, literal}` is not valid.

Writes persist to entity state, so a value written by one handler is readable as
`entity.<field>` in later handlers, not only the one that wrote it. Within a single handler,
guards and `expression` writes see `entity.*` as it stood before this handler's writes, while
the emit step runs after the writes and sees the new values.

## Emit

<ParamField path="emit" type="string or object">
  Publish the follow-up event. The object form `{event, fields, target, broadcast}` populates
  and routes it; the bare string form emits with an empty payload. A handler top-level `emit`
  is valid only when the handler has a single emit site.
</ParamField>

The emitted payload is producer-complete: `fields` must set every field the event declares,
each as a CEL expression (typically `entity.*` or `payload.*`). Nothing is copied from the
triggering event, so the bare string form is correct only for events with no payload fields.
The analyzer accepts a bare emit of a field-carrying event, so the empty payload surfaces only
at runtime; populate `fields` whenever the event carries data.

`target` and `broadcast` are a separate, routing-only concern: an intra-flow emit needs
neither, while a pin-declared output that crosses a flow boundary must set `target` (`sender`,
`{instance_id}`, `{flow, match}`, or `{flow, match, allow_fanout}`) or `broadcast: true`.

## List processing and computation

<ParamField path="accumulate" type="object">
  Wait for multiple arrivals. Fields: `into`, `expected_from`, `completion`
  (`all`, `threshold`, or `timeout`), `dedup_by` (default: sender session id), `on_complete`,
  `on_timeout`. The `dedup_by` default collapses multiple items from one sender; see
  [Accumulation and projection](/reference/accumulation) for that and for `materialize_from`.
</ParamField>
<ParamField path="compute" type="object">
  Run a computation primitive. `operation` is one of `weighted_average`, `pick_or_average`,
  `sum`, `min`, `max`, `count`; plus `tiers`/`keys`/`params` and `store_as`.
</ParamField>
<ParamField path="filter" type="object">
  Keep items matching a condition. Fields: `items_from`, `condition`, `store_as`.
</ParamField>
<ParamField path="reduce" type="object">
  Aggregate items to a single value. Same `operation` set as `compute`.
</ParamField>
<ParamField path="count" type="object">
  Count items matching a condition. Fields: `items_from`, `condition`, `store_as`.
</ParamField>
<ParamField path="fan_out" type="object">
  Emit one event per item. Fields: `items_from`, `target`, `emit`. Writes `fan_out.count` to
  handler context.
</ParamField>
<ParamField path="query" type="object or list">
  Pre-fetch cross-entity data. Fields: `entities`, `filter`, `group_by`, `count`, `select`,
  `store_as`. Runs before `clear_gates`.
</ParamField>
<ParamField path="clear" type="object">
  Reset accumulator or state buckets. `targets` is a list. Runs last.
</ParamField>

## Actions

<ParamField path="action" type="string or object">
  Invoke a platform action. One of `create_flow_instance`, `record_evidence`,
  `mailbox_write`, or `artifact_repo_commit`.
</ParamField>

- **`create_flow_instance`** requires `template`, `instance_id_from`, and `config_from`.
- **`record_evidence`** appends the payload to an accumulator; requires `evidence_target`
  (defaults to the node id).
- **`mailbox_write`** materializes a mailbox row; declared under `action.mailbox` with
  `item_type`, `severity` (`normal`/`urgent`/`critical`), `summary`, and an explicit
  `payload`.
- **`artifact_repo_commit`** commits allowlisted files to a local git repo and exposes a
  `swarm-artifact://` URL; declared under `action.artifact_repo`.

<Warning>
  Retired handler fields: `emits` (use `emit`), `payload_transform` (use `emit.fields`),
  `fan_out.emit_per_item` and `fan_out.emit_mapping` (use `fan_out.emit`). Selective
  `clear_gates: [gate]` is not current; gate clearing is all-or-nothing.
</Warning>
