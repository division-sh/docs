## Live spend tracking

Token usage and cost are tracked in the `spend_ledger`, one row per agent invocation, with
input tokens, output tokens, cost, model, and invocation type. Spend is attributable per
entity and per actor.

## Thresholds

Budget monitoring is configured in `policy.yaml` as percentages. Crossing one emits
`platform.budget_threshold_crossed` with a matching `level`:

```yaml policy.yaml
budget_warning_percent: 60
budget_throttle_percent: 80
budget_emergency_percent: 95
```

| Level | Meaning |
|---|---|
| `warning` | Spend is approaching the limit. |
| `throttle` | The runtime throttles further work. |
| `emergency` | The runtime escalates; a mailbox item is raised for a human. |

If these policy keys are absent, budget monitoring is disabled.

## Responding to an emergency

An emergency raises a [mailbox](/operate/mailbox) item so a human can approve continued spend,
pause the run, or stop it. Pausing ingress (`swarm control pause --all`) halts new work while
you decide.
