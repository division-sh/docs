## Events as the only communication primitive

Every interaction flows through events: an agent or node emits an event, the platform
validates and persists it, subscribers handle it, and new events continue the chain. Every
event has a typed payload schema in `events.yaml`, validated before the event is persisted.

Payloads are **producer-complete**: every declared field must be populated by the emit site's
`emit.fields`. There are no payload defaults and no implicit passthrough.

## Routing is derived, not chosen

`events.yaml` defines payload *shapes*, not routing. Routing comes from declared
subscriptions:

- `nodes.yaml` — `subscribes_to` and `event_handlers`
- `agents.yaml` — subscriptions and `emit_events`

Delivery follows from who subscribes: a subscribing node *owns* the event; a node and agents
together get *dual delivery*; a node alone is *node-only*; no node means *pure agent
delivery*. You change who receives an event by editing subscriptions, never by touching its
payload schema. No LLM decides routing.

## Addressing

References resolve by whether they contain a slash:

- **Local** — `order.created` — the thing of that name in the current flow.
- **Absolute** — `review/analysis/analysis.completed` — navigated from the root.

Subscriptions can use wildcards: `*` matches one path segment (any direct child flow), and
`**` matches any depth. Wildcards expand to include dynamic instances as they are created.

## The envelope/payload split

Runtime-owned fields live on the event *envelope*, read through `event.*`, never authored
into the payload. Pin-routed events carry route identities:

- `event.source` — the emitter's route identity (always set, never overwritten).
- `event.target` — a single recipient route identity.
- `event.target_set` — a concrete list of recipients for explicit fan-out.

A pin-declared output emit must resolve a target (`target: sender`, `{instance_id}`,
`{flow, match}`), a structural parent route, or `broadcast: true` — there is no silent
fallback to broadcast.

## Reserved metadata

Event-level metadata lives under the reserved `swarm:` namespace: `swarm.source` (external
producer proof), `swarm.consumer` (external sink), `swarm.status` (lifecycle, e.g.
`planned`), and the exceptional `swarm.producer`. The `platform.` event prefix is reserved
for the engine's own events.
